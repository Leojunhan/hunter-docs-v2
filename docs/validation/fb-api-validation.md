## 一、验证背景

Hunter 系统需要实现"发现目标"功能，核心需求：
- 爬取 FB 群组/帖子的评论
- 提取包含关键词的评论
- 获取目标用户信息

**验证目标：确认 FB Graph API 是否可行，或需要模拟登录方案。**

## 二、FB Graph API 调研结果

### 2.1 可用的 API 端点

| API 端点 | 功能 | 示例请求 |
|:--------|:----|:---------|
| `GET /{post-id}/comments` | 获取帖子的评论列表 | `GET https://graph.facebook.com/v25.0/{post-id}/comments` |
| `GET /{page-id}/posts` | 获取 Page 的帖子列表 | `GET https://graph.facebook.com/v25.0/{page-id}/posts` |
| `GET /{group-id}/feed` | 获取 Group 的帖子列表 | `GET https://graph.facebook.com/v25.0/{group-id}/feed` |

**API 版本：** v25.0（截至 2026-04）

---

### 2.2 权限要求

| 权限 | 说明 | 获取方式 |
|:----|:----|:--------|
| `pages_read_engagement` | 读取 Page 的评论和互动数据 | App 审核 + 用户授权 |
| `pages_read_user_content` | 读取用户在 Page 发布的内容 | App 审核 + 用户授权 |
| `Page Public Content Access` | 访问公开 Page 内容 | 特征权限，需额外申请 |
| User Access Token | 用户级别的访问令牌 | 用户授权后获取 |
| Group Access Token | 群组级别的访问令牌 | 群组管理员授权 |

---

### 2.3 API 调用示例

**获取帖子评论：**

```http
GET https://graph.facebook.com/v25.0/{post-id}/comments
    ?fields=message,from,created_time
    &limit=100
    &access_token={access_token}
```

**返回示例：**

```json
{
  "data": [
    {
      "message": "有没有人知道哪里可以借钱",
      "from": {
        "name": "John Doe",
        "id": "123456789"
      },
      "created_time": "2026-04-20T10:00:00+0000",
      "id": "comment-id"
    }
  ],
  "paging": {
    "next": "..."
  }
}
```

---

### 2.4 关键限制

| 限制 | 说明 | 影响 |
|:----|:----|:-----|
| **不能直接搜索帖子** | FB API 无搜索端点 | 需要先有 post-id 或 page-id |
| **不能爬取私人群组** | 只能爬取公开群组或用户已加入的群组 | 监控范围受限 |
| **需要权限审核** | App 需提交审核，说明用途 | 审核时间 1-2 周 |
| **速率限制** | 每小时约 600 次调用 | 高频爬取需控制频率 |

## 三、可行性评估

### 3.1 评估结论

| 场景 | FB Graph API | 模拟登录 | 建议 |
|:----|:------------|:--------|:----|
| **公开 Page 评论** | ✅ 可行 | ✅ 可行 | 优先 FB API |
| **公开 Group 评论** | ⚠️ 需验证 | ✅ 可行 | 双轨验证 |
| **私人群组评论** | ❌ 不可行 | ⚠️ 可行（需已加入） | 模拟登录 |
| **搜索关键词找帖子** | ❌ 不可行 | ✅ 可行 | 模拟登录 |

---

### 3.2 FB Graph API 可行性

| 评估项 | 结论 | 说明 |
|:------|:----|:----|
| **权限获取难度** | ⚠️ 中等 | 需提交 App 审核，说明用途 |
| **爬取范围** | ⚠️ 受限 | 只能爬取公开内容或用户授权的内容 |
| **稳定性** | ✅ 高 | 官方 API，合规稳定 |
| **成本** | ✅ 低 | 免费（审核通过后） |

**结论：FB Graph API 适合爬取公开 Page 内容，不适合爬取群组内容。**

---

### 3.3 模拟登录可行性

| 评估项 | 结论 | 说明 |
|:------|:----|:----|
| **技术难度** | ⚠️ 中等 | Playwright/Puppeteer 可实现 |
| **风险** | ⚠️ 高 | 可能被 FB 检测为机器人，封号 |
| **爬取范围** | ✅ 广 | 可爬取公开群组 + 已加入的私人群组 |
| **成本** | ⚠️ 中等 | 需维护账号池 |

**结论：模拟登录适合爬取群组内容，但有封号风险。**

## 四、技术方案对比

### 4.1 方案对比表

| 维度 | FB Graph API | 模拟登录 | 人工辅助 |
|:----|:------------|:--------|:--------|
| **合规性** | ✅ 高 | ❌ 低 | ✅ 高 |
| **稳定性** | ✅ 高 | ⚠️ 中 | ✅ 高 |
| **爬取范围** | ⚠️ 受限 | ✅ 广 | ⚠️ 受限 |
| **成本** | ✅ 低 | ⚠️ 中 | ⚠️ 高（人力） |
| **技术难度** | ⚠️ 中（权限审核） | ⚠️ 中（行为模拟） | ✅ 低 |
| **封号风险** | ✅ 无 | ❌ 高 | ✅ 无 |

---

### 4.2 建议方案

**双轨验证策略：**

| 验证轨道 | 内容 | 时间 |
|:--------|:----|:----|
| **轨道 A** | FB Graph API 权限申请 + 测试 | Week 1-2 |
| **轨道 B** | 模拟登录技术验证 + 账号池准备 | Week 1-2 |

**根据验证结果选择：**
- 如果 FB API 权限通过 → 优先使用 FB API
- 如果 FB API 权限不通过 → 使用模拟登录
