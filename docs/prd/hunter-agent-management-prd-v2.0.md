## 1. 需求背景

### 1.1 业务背景

Hunter 系统是**通用获客框架**，核心业务路径：

```
发现目标 → 清洗目标 → 触达目标 → 转化目标
```

**金融是第一个应用场景（OFW 海外劳工信贷），后续可扩展至教育、电商、服务等场景。**

核心目标：
- 在用户注意力出现的地方主动触达
- 实现触达→转化率 >15%
- 降低获客成本

Hunter 与 Copaw 的协作关系：

```
Hunter（管理界面）          Copaw（执行平台）
      ↓                         ↓
 配置策略 + 监控结果    ←→    执行任务 + 回传日志
```

**Hunter 是"策略配置中心 + 监控看板"，Copaw 是"执行引擎"。**

### 1.2 业务路径 → Agent + Skill 映射

| 业务路径 | Agent | Skill | 目的 |
|:--------|:------|:------|:-----|
| **发现目标** | 情报 Agent | FB评论爬取 Skill | 找到潜在用户 |
| **清洗目标** | 情报 Agent | NLP分类 + 规则评分 + 去重 Skill | 保证实时性和准确性 |
| **触达目标** | 执行 Agent | FB评论回复 Skill | 主动接触用户 |
| **转化目标** | 执行 Agent | 转化引导 Skill | 引导到转化点 |

**清洗后自动分配给执行 Agent + 剧本。**

### 1.3 用户痛点

| 用户角色 | 痛点 | 影响 |
|:---------|:-----|:-----|
| **运营人员** | 无法配置获客策略（监控目标、关键词） | 只能被动看结果，无法优化 |
| **运营人员** | 无法干预目标清洗和分配 | 自动分配可能不合理，无法干预 |
| **运营人员** | Agent 情报不互通，重复触达同一用户 | 用户体验差，资源浪费，可能被封号 |
| **管理人员** | 缺乏全局获客数据视图 | 无法评估效果，无法决策 |

### 1.4 预期收益

| 收益维度 | 预期效果 | 衡量指标 |
|:---------|:---------|:---------|
| **效率提升** | 运营人员可直接配置获客策略 | 配置周期从天级降至小时级 |
| **转化优化** | 清洗保证目标质量，提升转化效果 | 转化率提升 5%+ |
| **避免冲突** | Agent 情报共享，不重复触达 | 重复触达率 <1% |
| **可控性** | 管理人员可监控全局获客状态 | 问题发现时间分钟级 |

## 2. 需求描述

### 2.1 功能概述

Hunter Agent 管理模块是 Hunter 系统的第一阶段 MVP，核心功能：

- **Agent 管理** — 接入 Copaw 智能体，查看状态、配置参数
- **Skill 管理** — 管理 Agent 能力，承载截流配置（监控目标 + 关键词 + 人设 + 账号）
- **目标管理** — 管理截流目标用户，支持手动分配 Agent
- **执行日志** — 查看 Copaw 回传的任务执行记录
- **API 密钥** — 管理社媒平台 API Key

```
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 目标状态流转

```
发现 → 待分配 → 已分配 → 已触达 → 待回复 → 已转化 → 已关闭
  ↑                  ↑         ↑         ↑
  │                  │         │         │
  Copaw发现      手动/自动   Copaw执行  用户回复
  写入Hunter     分配Agent   回传结果   （或超时）
```

| 状态 | 说明 | 触发条件 |
|:-----|:-----|:---------|
| **发现** | 用户被发现，尚未处理 | Copaw 监控匹配到关键词 |
| **待分配** | 等待分配 Agent | 默认状态，或手动干预前 |
| **已分配** | 已指定 Agent 负责 | 自动分配或手动分配 |
| **已触达** | Agent 已执行触达 | Copaw 回传执行成功 |
| **待回复** | 等待用户回复 | 触达后等待用户响应 |
| **已转化** | 用户完成申请转化 | **依赖外部系统回传** |
| **已关闭** | 目标处理结束 | 手动关闭、用户拒绝、超时关闭 |

### 3.3 手动分配场景

| 场景 | 描述 | 触发时机 | 优先级 |
|:-----|:-----|:---------|:------:|
| **初次分配** | 新发现的目标，手动指定 Agent | 目标状态为"待分配"时 | P0 |
| **转交分配** | Agent A 太忙/能力不匹配，转给 Agent B | 运营人员主动操作 | P1 |
| **重新分配** | Agent A 触达失败，换 Agent B 再试 | 触达失败后 | P1 |

**分配原则：自动为主，人工为辅。**

### 3.4 异常流程

| 异常场景 | 处理方式 | 用户提示 |
|:---------|:---------|:---------|
| Agent 离线 | 分配时提示"Agent 离线，请选择其他 Agent" | Toast 提示 |
| 目标已被触达 | 分配时检测是否已被触达，避免重复 | 列表显示"已触达"状态 |
| API 密钥失效 | 执行失败，日志显示"API Key 失效" | 日志详情展示错误信息 |
| Copaw 连接失败 | Hunter 显示"执行平台离线" | Dashboard 告警 |

## A. AI 能力定义

### A.1 能力描述

Hunter Agent 管理模块涉及 AI Agent 的管理，AI 能力主要体现在：

- **Copaw 智能体** — 实际执行截流、对话等任务的 AI Agent
- **Hunter 管理界面** — 配置 AI Agent 的行为参数（人设、语气、策略）

Hunter 本身不运行 AI 模型，而是管理 Copaw 上的 AI Agent。

### A.2 能力边界

| 能力范围 | Hunter 支持 | Hunter 不支持 |
|:---------|:-----------:|:-------------:|
| Agent 状态监控 | ✅ | - |
| Skill 参数配置 | ✅ | - |
| 目标分配管理 | ✅ | - |
| 执行日志查看 | ✅ | - |
| AI 模型运行 | - | ❌（由 Copaw 负责） |
| Prompt 设计 | - | ❌（由 Copaw 负责） |
| 实际对话执行 | - | ❌（由 Copaw 负责） |

### A.3 Copaw 集成（待定）

|:-----|:-----|:----:|

| 心跳/超时配置 | Hunter 配置 → Copaw 执行 | **待定：Copaw 是否支持外部配置** |

## C. 效果指标

### C.1 核心效果指标

| 指标 | 定义 | 目标值 | 衡量方式 |
|:-----|:-----|:-------|:---------|
| **截流成功率** | 发现目标 → 触达成功比例 | >80% | 触达成功数 / 发现目标数 |
| **触达转化率** | 触达 → 申请转化比例 | >15% | 申请转化数 / 触达成功数 |
| **重复触达率** | 同一目标被多次触达比例 | <1% | 重复触达数 / 总触达数 |
| **Agent 利用率** | Agent 执行时间占比 | >60% | 执行时长 / 总时长 |
| **目标分配准确率** | 分配后成功触达比例 | >75% | 成功触达数 / 分配目标数 |

### C.2 测试集设计

| 测试场景 | 测试数据 | 验证指标 |
|:---------|:---------|:---------|
| **目标分配** | 100 个待分配目标 | 分配成功率 >95% |
| **情报共享** | 同一目标被 2 个 Agent 发现 | 重复触达率 <1% |
| **状态同步** | Agent 状态变化 | Hunter 显示延迟 <30s |
| **日志回传** | 100 条执行记录 | 日志完整率 >99% |

## D. 降级方案

### D.1 Copaw 离线降级

| 触发条件 | 降级策略 | 用户提示 |
|:---------|:---------|:---------|
| Copaw 连接失败 | Hunter 进入"离线模式"，显示历史数据 | "执行平台离线，正在重连..." |
| Copaw 离线 >5 分钟 | 停止自动分配，目标堆积在"待分配" | "执行平台离线，请等待恢复" |
| Copaw 离线恢复后 | 自动同步离线期间的日志和状态 | "执行平台已恢复" |

### D.2 Agent 故障降级

| 触发条件 | 降级策略 | 用户提示 |
|:---------|:---------|:---------|
| Agent 离线 | 目标分配时排除该 Agent | "Agent X 已离线，请选择其他 Agent" |
| Agent 执行失败率 >20% | 自动标记为"异常"，暂停分配 | "Agent X 执行异常，已暂停分配" |
| 所有 Agent 离线 | 目标堆积，告警通知管理员 | "所有 Agent 离线，请检查执行平台" |

### D.3 API 失效降级

| 触发条件 | 降级策略 | 用户提示 |
|:---------|:---------|:---------|
| 单个 API 失效 | 使用该 API 的 Skill 自动禁用 | "Skill X 的 API 失效，已暂停" |
| 所有 API 失效 | 停止所有截流，告警通知 | "所有 API 失效，请检查密钥配置" |
| API 即将过期 | 提前 7 天告警提醒 | "API Key 将于 7 天后过期" |

### D.4 目标堆积降级

| 触发条件 | 降级策略 | 用户提示 |
|:---------|:---------|:---------|
| 待分配目标 >100 | 提高自动分配速度，告警提醒 | "目标堆积，建议增加 Agent" |
| 待分配目标 >500 | 暂停发现，优先处理堆积 | "目标堆积严重，暂停发现" |
