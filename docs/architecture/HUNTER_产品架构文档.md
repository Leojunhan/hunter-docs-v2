## 零、核心设计原则

| 原则 | 说明 |
|------|------|
| **Agent 永生** | 智能体永远存在，不依赖特定剧本 |
| **剧本=马甲** | 1对1，人设+账号+话术绑定，剧本失效（账号被封）→Agent 切换新剧本 |
| **Skill=能力层** | Skill 定义 Agent 能做什么（截流、爬取、对话） |
| **目标=截流闭环核心** | 目标是截流的起点和终点，没有目标管理，截流闭环不完整 |
| **Hunter=管理界面，Copaw=执行平台** | Hunter 配置+监控，Copaw 执行+回传 |

      │                          │

└─────────────┘            └─────────────┘
```

**核心一句话：Hunter 配置下发 + Copaw 执行回传 = 完整协作**

### 2.2 Hunter 模块职责

| Hunter 模块 | 职责 | Copaw 交互 |
|-------------|------|-----------|
| **Agent 管理** | Agent 状态监控、剧本绑定 | 接收 Agent 状态、下发剧本绑定 |
| **Skill 管理** | Skill 配置（情报/执行/风控） | 下发 Skill 配置 |
| **目标管理** | 目标状态、分配管理 | 接收发现的目标、下发分配指令 |
| **执行日志** | 执行记录追踪 | 接收执行日志 |
| **Dashboard** | 数据概览、告警通知 | 接收告警 |
| **API 密钥** | 社媒平台密钥管理 | 为 Copaw 提供凭证

### 2.3 Agent 类型

| Agent 类型 | 职责 | 运行位置 |
|-----------|------|---------|
| **情报 Agent** | 爬取采集、NLP 分类、目标发现 | Copaw |
| **执行 Agent** | API 调用、内容生成、触达执行 | Copaw |
| **风控 Agent** | 状态检测、阈值判断、告警触发 | Copaw |

**说明：Agent 类型由 Copaw 定义，Hunter 只做状态监控和剧本绑定。**

### 2.5 剧本切换流程

```
剧本失效（账号被封）
    │
    ▼
① 检测剧本状态
   Copaw 风控 Agent 检测账号被封
    │
    ▼
② 上报剧本失效
   回传 Hunter：剧本 X 失效
    │
    ├── 自动切换路径
    │   │
    │   ▼
    │   ③ 查找备用剧本
    │      Agent X 预设的备用剧本列表
    │       │
    │       ▼
    │   ④ 自动切换
    │      选择可用剧本，立即切换
    │       │
    │       ▼
    │   ⑤ 继续执行
    │      Agent X 使用新剧本继续截流
    │
    └── 手动切换路径
        │
        ▼
    ③ 告警通知
        Hunter 告警：剧本 X 失效，请手动切换
         │
         ▼
    ④ 运营干预
        运营人员在 Agent 管理手动分配新剧本
         │
         ▼
    ⑤ 切换完成
        Agent X 使用新剧本继续截流
```

    │
    ▼

   └── 话术：开场白"老乡你好..."
    │
    ▼
⑤ Agent 执行触达
   Copaw 执行 Agent 执行触达
   ├── 使用马甲A账号
   ├── 人设：热心老乡
   ├── 私信 @john
   └── 内容：开场白话术
    │
    ▼
⑥ 执行记录回传
   Copaw 回传执行结果
   ├── Hunter 执行日志记录
   ├── 状态：成功/失败
   └── 错误信息（如有）
    │
    ▼
⑦ 目标状态更新
   Hunter 目标管理状态更新
   ├── 成功 → 状态：已触达 → 待回复
   └── 失败 → 状态：待分配（重新分配）
    │
    ▼
⑧ 用户回复处理
   用户 @john 回复
   ├── Agent X 继续对话
   ├── 使用跟进话术
   └── 引导转化
    │
    ▼
⑨ 转化结果回传
   外部系统（借贷审批）回传
   ├── 用户完成申请
   ├── Hunter 目标管理状态更新：已转化
   └── Agent 记忆更新：用户偏好、转化记录
    │
    ▼
⑩ 数据回流
   数据回流优化
   ├── Skill 效果评估更新
   ├── 剧本消耗标记
   ├── 目标转化记录
   └── 风控模型优化
```

## 五、数据流闭环

```
┌─────────────────────────────────────────────────────────────────┐
│                      数据流闭环                                   │
└─────────────────────────────────────────────────────────────────┘

         ┌───────────────────────────────────────────────────┐
         │                   数据回流                         │
         │                                                   │
         ▼                                                   │
┌─────────────┐   ┌─────────────┐   ┌─────────────┐         │
│   Agent     │──→│   Skill     │──→│    目标     │─────────┤
│  (能力+记忆) │   │ (截流参数)  │   │  (截流对象)  │         │
│             │   │             │   │             │         │
│  记忆更新    │   │ 效果评估    │   │ 状态流转    │         │
└─────────────┘   └─────────────┘   └─────────────┘         │
       ▲                 │                 │                 │
       │                 │                 │                 │
       │                 │                 │                 │
       │                 ▼                 ▼                 │
       │          ┌─────────────┐   ┌─────────────┐         │
       │          │    任务     │──→│  执行记录   │─────────┤
       │          │ (Copaw执行) │   │ (结果追踪)  │         │
       │          │             │   │             │         │
       │          │  执行实例   │   │  结果回传   │         │
       │          └─────────────┘   └─────────────┘         │
       │                                   │                 │
       │                                   │                 │
       │                                   │ 回流            │
       │                                   ▼                 │
       │          ┌─────────────┐   ┌─────────────┐         │
       └───←─────│    剧本     │←──│  转化结果   │←────────┘
  Phase 2      │  (马甲)     │   │ (外部系统)  │
               │             │   │             │
               │  消耗标记   │   │  回传状态   │
               └─────────────┘   └─────────────┘

核心：目标 + 执行记录是数据回流的双枢纽
  → 目标状态追踪（截流闭环）
  → 执行记录回流（效果优化）
  → Agent 记忆更新（能力进化）
```

## 六、Phase 划分说明

### 6.1 Phase 1 MVP（核心闭环）

**目标：实现截流闭环，验证单位经济**

| 模块 | 功能 | 说明 |
|------|------|------|
| Dashboard | 数据概览 | Agent 数量、目标数、触达数、转化率 |
| Agent 管理 | Agent 接入 | Copaw Agent 状态监控、剧本绑定 |
| Skill 管理 | 截流参数 | 监控目标、关键词、人设、账号（暂代） |
| 目标管理 | 目标流转 | 状态流转、手动分配、分配记录 |
| 执行日志 | 结果追踪 | 执行记录展示、错误分析 |
| API 密钥 | 平台接入 | FB/WhatsApp/Meta API 管理 |

### 6.2 Phase 2 扩展（辅助功能）

**目标：优化截流效率，提升转化率**

| 模块 | 功能 | 说明 |
|------|------|------|
| 剧本管理 | 剧本配置 | 人设+账号+话术，从 Skill 拆分 |
| 任务管理 | 特殊干预 | 手动创建任务、任务干预 |
| 数据报表 | 效果分析 | 转化分析、趋势分析、导出报表 |
| 配置中心 | 全局配置 | 关键词规则、采集设置、自动化规则 |
| 权限管理 | 安全增强 | 角色管理、权限配置 |
| 告警中心 | 告警管理 | 告警规则、告警历史 |

### 6.3 Phase 3 探索（高级功能）

**目标：自动化、智能化、开放化**

| 模块 | 功能 | 说明 |
|------|------|------|
| 竞品管理 | 竞品分析 | 竞品监控、策略建议 |
| 自动化 | 自动规则 | 自动分配、自动切换、自动关闭 |
| API 开放 | 第三方接入 | 数据开放、事件推送 |

| 待定项 | 问题描述 | 影响范围 | 决策方 |
|:-------|:---------|:---------|:-------|
